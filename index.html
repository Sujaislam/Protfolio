02Architecture Philosophy

Studies in Odoo module design

Two basic paradigms that shape how I implement custom modifications to the Odoo ERP framework.

A.01

Inheritance Over Overrides

I extend Odoo modules strictly using class inheritance (`_inherit`). This guarantees standard features are preserved, avoiding core modifications and keeping migration pathways open.

A.02

Isolated Sync Middleware

API synchronization triggers should not block database operations. I implement decoupled background queue tasks (Odoo cron) to manage integrations with external platforms.
